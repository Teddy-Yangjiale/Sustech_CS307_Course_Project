## Task 2 Query Processing

### Basic DDL

实现内容：
- `show tables`
- `describe table`
- `drop table`
- `explain select ...`
- `alter table add column ...`
- `alter table drop column ...`

说明：
- `show tables` 和 `describe table` 在 `LogicalPlanner` 中手动识别。
- `drop table` 会删除元数据、数据文件、表目录，并清理 BufferPool。
- `alter table` 会重写表数据文件，保证新增/删除列后还能继续 insert/select。

### Basic Query / Update / Delete

实现内容：
- 任意列投影，例如 `select t.id, t.name from t`。
- 不带表名前缀的列，例如 `select id from t`。
- `select *`。
- `WHERE` 支持：
  - `AND`
  - `OR`
  - `=`
  - `>`
  - `>=`
  - `<`
  - `<=`
  - 字面量 `IN`
- `insert` 支持多行 values。
- `update` 支持按条件更新。
- `delete from t where ...` 支持条件删除。
- 修正 CHAR 定长序列化，避免字符串字段影响后续列偏移。

### Aggregate / Order / Join / Subquery

实现内容：
- `count(*)`
- `max(...)`
- `min(...)`
- 多聚合 select list。
- `group by`
- 多列 `order by`
- `nested loop join`
- `IN (select ...)`
- `EXISTS (select ...)`
- 常见相关子查询，例如：

```sql
select students.id
from students
where exists (
  select scores.id from scores where scores.id = students.id
);
```

说明：
- Join 使用内存物化 nested loop join。
- Order By 使用内存排序。
- Aggregate 使用内存分组聚合。
- 子查询目前覆盖课程项目常见形式，不做代价优化。

## Task 3 Index

实现内容：
- 支持 `create index index_name on table_name(column_name)`。
- 支持 `drop index index_name`。
- 索引元数据会写入 `CS307-DB/meta/meta_data.json`，每个索引记录 indexName、tableName、columnName 和 BTREE 类型。
- `DBManager` 维护运行期内存索引表，启动后按元数据懒加载/重建索引。
- 实现内存 B+ tree，支持节点分裂、等值查询、范围查询、重复 key 对应多个 RID，以及按层打印节点。
- 支持同一张表多个单列索引。
- 创建索引时扫描已有数据建树。
- 插入、删除、更新会同步维护相关索引。
- 事务回滚或回滚到 savepoint 后会从磁盘状态重建索引。
- 简单谓词会走索引扫描：
  - `col = value`
  - `col > value`
  - `col >= value`
  - `col < value`
  - `col <= value`

说明：
- 索引本体按 Project2 PDF 要求只保存在内存中，持久化的是索引元数据。
- 复杂 `AND` / `OR` 条件仍保守使用原有顺序扫描和过滤逻辑，避免改变查询语义。
- `IndexScanOperator` 通过索引返回 RID，再从记录文件读取正常 `TableTuple`。
