## Task 2 Query Processing

### Basic DDL

实现内容：
- `show tables`
- `describe table`
- `drop table`
- `explain select ...`
- `alter table add column ...`
- `alter table drop column ...`

说明：
- `show tables` 和 `describe table` 在 `LogicalPlanner` 中手动识别。
- `drop table` 会删除元数据、数据文件、表目录，并清理 BufferPool。
- `alter table` 会重写表数据文件，保证新增/删除列后还能继续 insert/select。

### Basic Query / Update / Delete

实现内容：
- 任意列投影，例如 `select t.id, t.name from t`。
- 不带表名前缀的列，例如 `select id from t`。
- `select *`。
- `WHERE` 支持：
  - `AND`
  - `OR`
  - `=`
  - `>`
  - `>=`
  - `<`
  - `<=`
  - 字面量 `IN`
- `insert` 支持多行 values。
- `update` 支持按条件更新。
- `delete from t where ...` 支持条件删除。
- 修正 CHAR 定长序列化，避免字符串字段影响后续列偏移。

### Aggregate / Order / Join / Subquery

实现内容：
- `count(*)`
- `max(...)`
- `min(...)`
- 多聚合 select list。
- `group by`
- 多列 `order by`
- `nested loop join`
- `IN (select ...)`
- `EXISTS (select ...)`
- 常见相关子查询，例如：

```sql
select students.id
from students
where exists (
  select scores.id from scores where scores.id = students.id
);
```

说明：
- Join 使用内存物化 nested loop join。
- Order By 使用内存排序。
- Aggregate 使用内存分组聚合。
- 子查询目前覆盖课程项目常见形式，不做代价优化。
